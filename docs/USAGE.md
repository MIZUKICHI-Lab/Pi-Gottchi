# 使い方

## 基本操作

| 操作 | 反応 |
|---|---|
| 起きている間に話しかける | Gemini Live API が発話を検出し、音声と字幕で返答 |
| ボタンを1回押す | なでられて喜ぶ |
| ボタンを2回押す | 軽い揺れを模擬 |
| ボタンを3回以上押す | 強い揺れを模擬 |
| Live 接続中以外にボタンを押しながら話す | ボタンを離した後に REST API で処理 |
| ボタンを10秒間押す | 状態を保存してシャットダウン |
| 本体を揺らす | 揺れの強さに応じた反応（対応センサー接続時） |
| 充電器を挿す | 食事を模した反応、バッテリー残量表示 |
| 「おやすみ」「さよなら」「またね」等を直接伝える | 返答後にアプリ内スリープ |

2回・3回押しは、加速度センサーなしで表情や音を確認するためにも利用できます。

## 会話状態

RGB LED と表情は、現在の状態を示します。

| 状態 | 主なLED色 | 意味 |
|---|---|---|
| 聞き取り | 青 | ユーザーの発話を受け取っている |
| 考え中 | オレンジ | 返答を待っている |
| 発話 | 緑 | 音声を再生している |
| 睡眠 | 暗い紺 | マイクの Live 送信を停止している |
| 強い揺れ | 赤 | 目を回している |

スピーカーの音をマイクが拾い続けないよう、「もこ」が話している間はマイク入力を破棄する半二重方式です。返答中に話しかけても、その発話は処理されません。

## 睡眠と起床

通常は最後の操作・会話から20秒、22時から翌7時までは10秒で眠ります。自動睡眠中もLive会話は待機し続け、話しかけるとそのまま起きて返答します。「おやすみ」「もう寝るね」「さよなら」「バイバイ」「またね」のような直接的な発話では、もこの返答が終わってからマイクも止める深い睡眠へ入ります。「おやすみって何」のような言及はコマンドにしません。

自動睡眠中は睡眠顔とサーボの睡眠姿勢へ移りますが、Liveセッションと連続録音は維持します。明示的な深い睡眠ではLiveも停止します。Pi Zero Wでは既定で睡眠顔を表示したままにします。音声誤認識からOSの電源は切りません。

次の操作で起きます。

- ボタンを押す
- 対応センサーを接続した状態で本体を揺らす
- 一定以上の大きさの音を入力する

音による起床確認は3秒間隔なので、確実に起こす場合はボタンまたは対応センサーを使ってください。サーボ連動の設定は[ハードウェア連動](HARDWARE_INTEGRATION.md)を参照してください。

## 記憶

会話履歴は `app/memory.json` に保存されます。眠りに入ると、直近の会話を Gemini で短い長期記憶へ整理します。保存対象には名前、好み、約束、最近の話題などが含まれることがあります。

記憶を消すには、サービスを停止してからファイルを削除します。

```bash
sudo systemctl stop moko.service
rm app/memory.json
sudo systemctl start moko.service
```

実際の配置先に応じてパスを変更してください。

## 状態データ

気分や親密度などの端末内状態は `app/state.json` に保存されます。初期化する場合は、記憶と同様にサービス停止中に削除してください。

`memory.json`、`state.json`、生成済み音声の `voices/` は Git の管理対象外です。
