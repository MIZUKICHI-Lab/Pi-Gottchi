# アーキテクチャ

## 全体像

```text
Whisplay HAT
  ├─ LCD / RGB LED ───────────────┐
  ├─ ボタン ─────────────────────┤
  ├─ マイク ─→ ALSA ─→ Gemini ──┤→ chara.py の状態管理
  └─ スピーカー ← ALSA ← 音声 ──┤       │
                                  └───────┴→ face.py の描画
任意センサー ─→ I2C ─→ imu.py ──────────┘
PiSugar ──────→ I2C ─→ 残量・充電状態 ──┘
確定発話 ─→ intents.py ─→ アプリ内スリープ ─┐
リアクション ─→ ServoAnimator ─→ PWM/SG90 ─┘
```

`chara.py` がイベント、状態、描画、音声再生をまとめるメインループです。ネットワーク処理、録音、加速度監視は別スレッドまたは子プロセスで動きます。

## ファイルの役割

| ファイル | 役割 |
|---|---|
| `app/chara.py` | メインループ、状態遷移、ボタン、睡眠、バッテリー |
| `app/face.py` | PIL による顔・字幕・アニメーション描画 |
| `app/live.py` | Gemini Live API、連続録音、VAD、音声ストリーム再生 |
| `app/intents.py` | 確定発話から安全な端末内コマンドだけを判定 |
| `app/voice.py` | REST 会話、TTS、定型音声キャッシュ、ローカル音声代替 |
| `app/memory.py` | 会話ターンの保存と長期記憶への要約 |
| `app/voiceprint.py` | MFCC ベースの簡易音声特徴比較 |
| `app/imu.py` | I2C センサー検出と揺れイベント生成 |
| `app/servo.py` | Linux sysfs PWMと非同期リアクション動作 |
| `app/splash.py` | アプリ起動前のスプラッシュ表示 |

## 音声対話の流れ

通常は Gemini Live API を使用します。

```text
arecord (16 kHz / mono / PCM)
  → 100msチャンク（設定可能）
  → WebSocket
  → Gemini の発話検出・応答生成
  → 24 kHz PCM 音声
  → aplay
```

入力と出力の文字起こしを受け取り、LCD の字幕と `memory.json` の会話履歴にも利用します。文字起こしは独立配送で順序保証がないため、マイク停止や返答音声の再生可否には使いません。応答再生中とサーボ動作中は録音データを破棄し、スピーカー音・機械音の自己入力を抑えます。詳細は[音声パイプライン設計](AUDIO_PIPELINE.md)を参照してください。

Live 接続を利用できない間は、ボタンを押している間の音声を WAV に録音し、REST API で文字・音声応答を生成します。API が利用できない場合でも、画面、ボタン、電子音、センサーなどローカル機能は動作します。

## 接続管理

Live 接続には次の復旧処理があります。

- 無音時の ping による接続確認
- 切断後3秒から始まる指数バックオフ再接続
- サーバーからの切断予告への対応
- 送信エラー時の即時切断・再接続
- 長いマイク停止時の `audioStreamEnd`
- 再生直後の録音チャンク破棄によるエコー抑制

## ハードウェアイベント

`chara.py` は表情、音声、LED、バックライト、サーボを同じ名前付きイベントから更新します。サーボは専用スレッドで動き、最新イベントだけを安全角度内で処理します。音声コマンドによる睡眠はOS shutdownではなく、Live停止・画面減光・睡眠姿勢をまとめた可逆なアプリ状態です。詳細は[ハードウェア連動](HARDWARE_INTEGRATION.md)を参照してください。

## 保存ファイル

| パス | 内容 | Git管理 |
|---|---|---|
| `app/.env` | APIキーと環境設定 | 対象外 |
| `app/memory.json` | 会話と長期記憶 | 対象外 |
| `app/state.json` | 気分・親密度など | 対象外 |
| `app/voices/` | 生成した定型音声 | 対象外 |
| `/tmp/moko_*.wav` | 一時録音・再生音声 | 一時ファイル |

秘密情報や個人情報を含む可能性があるため、これらを Issue やログへ添付する前に内容を確認してください。
