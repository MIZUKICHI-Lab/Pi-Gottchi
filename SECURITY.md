# セキュリティとプライバシー

## 脆弱性の報告

APIキーの漏えい、任意コマンド実行、認証回避など、公開前に対応が必要な問題は公開 Issue に詳細を書かないでください。GitHub リポジトリの Security タブに Private vulnerability reporting が表示される場合は、そこから報告してください。有効でない場合は、機密情報を含めずにリポジトリ所有者へ非公開の連絡手段を確認してください。

## API キー

- Gemini API キーは `app/.env` に保存し、権限を `600` にしてください。
- `.env` を Git、Issue、スクリーンショット、診断ログへ含めないでください。
- 漏えいした可能性があるキーは Google AI Studio で直ちに無効化し、新しいキーへ交換してください。
- 可能であれば API 側で利用制限、予算通知、使用量監視を設定してください。

## 音声と会話データ

起きている間、マイク音声は Gemini Live API へ送信されます。会話の文字起こしと要約された記憶は端末内の `app/memory.json` に保存されます。systemd の journal にも会話内容が記録される場合があります。

この端末を第三者がいる場所で使う場合は、録音・外部送信について事前に知らせ、地域の法令や施設の規則に従ってください。秘密情報、個人情報、第三者の音声を不用意に扱わないでください。

Google に送信されたデータの保存・利用条件は、契約、APIプラン、地域、規約変更により異なります。Google の最新の Gemini API 利用規約とプライバシー情報を確認してください。

## 端末内データを消去する

サービス停止後、必要に応じて次を削除します。

```bash
sudo systemctl stop moko.service
rm -f app/memory.json app/state.json
rm -rf app/voices
```

実際のインストール先に合わせてパスを変更してください。`voices/` を削除すると、次回オンライン起動時に定型音声が再生成され、API利用量が発生する場合があります。

journal の会話ログも消去する必要がある場合は、OS全体の監査・保守方針を確認してから systemd journal の保持設定を変更してください。

## 声紋ライトの制限

`VOICEPRINT=gate` は簡易的な類似度判定です。認証、アクセス制御、本人確認には使用しないでください。録音条件による誤拒否・誤受理が起こり得ます。

## ハードウェアの安全

- GPIO とカメラの配線は必ず電源OFFで変更してください。
- 3.3V 専用センサーを5Vへ接続しないでください。
- サーボには外部5V電源を推奨し、Raspberry Pi とGNDを共通にしてください。
- 外部サーボ電源のプラスをRaspberry Piの5Vへ接続せず、信号の安全角度を校正してから `SERVO_ENABLED=1` にしてください。
- 露出した導体や PiSugar のポゴピン周辺は絶縁し、短絡を防いでください。

音声コマンドは誤認識し得るため、OS shutdownや任意GPIO操作へ直接結び付けません。汎用AIエージェントへroot shellや無制限のGPIO権限を与えず、端末内で許可した可逆な操作だけを実行してください。

詳しい配線上の注意は[配線ガイド](app/WIRING.md)を参照してください。
